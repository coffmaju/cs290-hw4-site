Overview

In this assignment, you will redesign your restaurant ordering page to make it more user-friendly based on good web development practices. In real software development, usability matters as much as functionality. You are now improving your website to make it easier for customers to build, preview, and submit their order.

In HW4, you will enhance your Byte Bistro website by creating an interactive ordering experience using JavaScript. You will also connect the order form to menu-order.php by submitting a JSON version of the order.

Requirements

  • A menu and order form together on a single page
  • Clickable entrée, dessert, and beverage menu items
  • Quantity input appears next to each selected item
  • A Live Order or Selected Items area organized by entrée, beverage, and dessert sections
  • A Preview Receipt button
  • Receipt printed to the console for the developer view
  • Receipt printed into the DOM for the customer view
  • Receipt includes subtotal, tax, tip, delivery fee, and total
  • Ability to update the order after previewing; the order should not be locked after the first preview
  • Final order is submitted to the server as JSON
  • menu-order.php displays a confirmation page and calculates totals using the submitted order data

Customers will be able to:

  • Select items by clicking on menu cards (Activity 9)
  • Specify quantities and see selected items in a sidebar (Activity 10)
  • Generate a structured JavaScript order object and display a formatted receipt (Activity 11)
  • Submit the order using a standard HTML form along with a JSON version of the order (Activity 12)

This assignment builds on your work from HW2 and HW3 and includes Activities 9, 10, 11, and 12.

What You’ll Implement

1) Activity 9: Clickable Menu Items

  • Create or update menu-order.html with a grid layout
  • Create clickable menu cards for entrées, beverages, and desserts
  • Clicking a menu card toggles selection and shows the quantity input and action buttons
  • Use data-name, data-price, and data-category attributes
  • Log item name, category, price, and quantity to the console while testing

2) Activity 10: Live Order Sidebar

  • Add a live-updating sidebar that shows item name and quantity as the user builds the order
  • Group sidebar items by category: entrées, beverages, and desserts
  • Create arrays to track selected items
  • Each selected item should store its name, price, and quantity
  • Allow users to confirm, update, or remove selected items
  • Move or adapt the order form so it appears with the sidebar

3) Activity 11: Build Order Object + Receipt Preview

  • Build a structured JavaScript order object using the HW3 order format
  • Include customer name, date, city, delivery status, tip percent, entrées, drinks, desserts, and add-ons
  • Update priceMap so it matches the item names and prices on your current menu
  • Reuse your process-orders.js functions from HW3
  • Modify printReceipt() so it builds one receipt string, logs it to the console, and returns the string
  • Display the returned receipt string inside the page using <pre id="receiptOutput"></pre>
  • Receipt must include subtotal, tax, tip, delivery fee, and total

4) Activity 12: Submit Order JSON to PHP

  • Add hidden inputs for orderJson and taxRate
  • Use JSON.stringify(order) to place the order object into the hidden input before submission
  • Use taxRates[order.city] to place the selected city’s tax rate into the hidden input
  • Submit the form to menu-order.php
  • Use target="_blank" so the PHP confirmation page opens in a new tab for comparison
  • Use the provided generic menu-order.php file
  • Add your own stylesheet links, header, navigation bar, and footer to menu-order.php so it matches the rest of your site
  • Compare the JavaScript receipt preview with the PHP confirmation page

Important: The provided menu-order.php file handles the PHP processing logic. Your job is to send the correct data and style the confirmation page so it fits your Byte Bistro site.
