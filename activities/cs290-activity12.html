Activity 12 – Submit the Order JSON to PHP

In Activity 11, you built an order object and previewed a receipt on your page. In this activity, you will submit that order to menu-order.php so the server can process it.

Big idea: Your JavaScript shows a preview for the user. The PHP page receives the same data and calculates the total again so you can compare the results.

What You Will Build

  • A hidden input that stores the full order object as JSON
  • A hidden input that stores the selected city’s tax rate
  • A submit event handler that fills both hidden inputs
  • A confirmation page (menu-order.php) that displays the result

Step 1: Update the Form Action

Your form should now submit to menu-order.php instead of order.php.

<form id="order-form" action="menu-order.php" method="post" target="_blank">

Note: Using target="_blank" opens the PHP confirmation page in a new tab so you can compare it with your receipt preview.

Step 2: Add Hidden Inputs

In HTML forms, the <input type="hidden"> element allows you to include data that is submitted to the server but is not visible or editable by the user in the browser window. It requires a name and a value to be useful. JavaScript can dynamically read or change the value of the hidden inputs, making them useful for capturing client-side data.

The order object exists in JavaScript. To submit it with the form, convert it to JSON and store it in a hidden input.   

You will also send the tax rate based on the selected city:

taxRates[order.city]

Add these inside your form:

<input type="hidden" name="orderJson" id="orderJson" value="[]">
<input type="hidden" name="taxRate" id="taxRate" value="0">

Step 3: Fill Hidden Inputs on Submit

When the user clicks the submit button in the form with id="order-form", the order object is built and stored in the hidden input orderJson. The tax rate for the selected city is stored in the hidden input taxRate.

This ensures the hidden fields contain the most current order selections before the form is submitted. After the event listener finishes running, the form submission sends all form field values to the server. 

document.getElementById("order-form").addEventListener("submit", function () {

  const order = buildOrderObject();

  // Fill hidden inputs
  document.getElementById("orderJson").value =
      JSON.stringify(order);

  document.getElementById("taxRate").value =
      taxRates[order.city];

});

This activity uses two separate actions: previewing the order and submitting the order.

Preview Button:
  • Uses JavaScript only
  • Displays the receipt on the page
  • Does NOT send data to the server
Submit Button:
  • Sends data to PHP
  • Uses hidden inputs
  • Opens a new page or tab with the confirmation

Step 4: Use the Provided menu-order.php File

You are given a working menu-order.php file.

This file already:

  • Receives the submitted order JSON
  • Decodes the JSON into a PHP array
  • Calculates subtotal, tax, tip, delivery fee, and total

Important: You are NOT expected to write the PHP calculation logic.

Also important: The order JSON already includes item prices, so the PHP file uses those values directly.

Step 5: Compare the Results

Test your project:

  1. Select items and options
  2. Click Preview Receipt
  3. Click Submit
  4. Compare the two results

Compare:

  • Total

The values on the PHP page should match your JavaScript receipt preview.
