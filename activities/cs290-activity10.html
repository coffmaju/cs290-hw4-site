Activity 10 – Live Order Sidebar and Order Form

In Activity 9, you created clickable menu items and logged selections to the console. In this activity, you will begin tracking selected items and displaying them in a Live Order sidebar.

You will also move and update your existing order form so it works with your interactive menu.

Important: In this activity, you are building the structure of the order. Do not calculate totals or display a receipt yet. That will be done in Activity 11.

Step 6: Update Confirm Behavior

In Activity 9, the confirm button logged the selected item to the console. Now the confirm button should also prepare the item to be stored in the correct array.

First, make sure your confirm button event listener is still connected:

item.querySelector('.confirm-btn').addEventListener('click', confirmSelectedItem);

When the confirm button is clicked, the browser calls confirmSelectedItem and automatically passes in an event object.

function confirmSelectedItem(event) {
  event.stopPropagation();

  const item = event.target.closest('.menu-item');

  const name = item.dataset.name;
  const price = parseFloat(item.dataset.price);
  const category = item.dataset.category;

  const qtyInput = item.querySelector('.qty-input');
  const quantity = parseInt(qtyInput.value);

  // TODO: Validate quantity
  // Hint:
  // If quantity is not a number OR less than 1,
  // log a message and stop the function using return.

  const selectedItem = {
    name: name,
    price: price,
    quantity: quantity
  };

  updateSelectedArray(category, selectedItem);
  updateLiveOrder();

  console.log(`Confirmed: ${name} x ${quantity}`);
}

Validation hint: Use isNaN(quantity) to check whether the quantity is not a number. Also check whether quantity < 1. If the quantity is invalid, use console.log(...) and then return;. The return statement stops the function early so the invalid item is not added to the order.

Why do we use event.stopPropagation()? The confirm button is inside the menu item. Without this line, clicking Confirm may also trigger the menu item's click event again.

Step 8: Display the Live Order

Create a function that updates the three sidebar lists. This function should be called after an item is confirmed or removed.

function updateLiveOrder() {
  updateList("entree-list", entrees);
  updateList("beverage-list", beverages);
  updateList("dessert-list", desserts);
}

function updateList(listId, items) {
  const list = document.getElementById(listId);
  list.innerHTML = "";

  items.forEach(function(item) {
    const li = document.createElement("li");
    li.textContent = `${item.name} x ${item.quantity}`;
    list.appendChild(li);
  });
}

Note: Setting list.innerHTML = "" clears the old list before rebuilding it. This keeps the sidebar from showing duplicate or outdated items.

Step 9: Update Remove Behavior

When the remove button is clicked, remove the item from the correct array and update the Live Order sidebar.

function removeSelectedItem(event) {
  event.stopPropagation();

  const item = event.target.closest('.menu-item');

  const name = item.dataset.name;
  const category = item.dataset.category;

  removeFromSelectedArray(category, name);

  const qtyInput = item.querySelector('.qty-input');
  const confirmBtn = item.querySelector('.confirm-btn');
  const removeBtn = item.querySelector('.remove-btn');

  qtyInput.value = "";
  qtyInput.style.display = "none";
  confirmBtn.style.display = "none";
  removeBtn.style.display = "none";

  item.classList.remove("selected");

  updateLiveOrder();

  console.log(`Removed item: ${name}`);
}
