Activity 11 – Build Order Object + Full Receipt Preview

Time: 2 class periods

In Activity 10, you built an interactive menu where users select items and see them appear in a Live Order sidebar.

In this activity, you will connect that work to your HW3 code. You will build an order object from the webpage and reuse your existing functions to compute and display a full receipt.

Big Idea:
Activity 10 = what the user selected (data)
Activity 11 = compute and display results from that data

Step 1: Load Both JavaScript Files

Your page now uses two JavaScript files:

  • process-orders.js → contains your HW3 calculation functions

  • menu-interact.js → contains your menu and form logic

<script src="js/menu-interact.js" defer></script>

Step 4: Helper Functions

These functions collect small pieces of data that you will use later.

function getTodayDateString() {
  const today = new Date();
  return today.toISOString().split("T")[0];
}

This converts today's date into the same format used in HW3.

function getSelectedAddOns() {
  const addOnsList = document.querySelectorAll('input[name="addOns"]:checked');
  const addOns = [];

  addOnsList.forEach(function(item) {  
    addOns.push({
      name: item.value,
      price: parseFloat(item.dataset.price),
      quantity: 1
    });
  });

  return addOns;
}

This collects all checked add-ons into an array of objects.

Important: Each add-on checkbox must include a data-price attribute. If data-price is missing, your program will produce NaN values and your totals will be incorrect.

Example:

<input type="checkbox" name="addOns" value="Avocado" data-price="1.25">

Note: Add-ons are stored with a quantity of 1. This keeps the data structure consistent with other menu items. In a more advanced version of this project, users could select quantities for add-ons as well.

Important: Add-ons are now stored as objects with name, price, and quantity.
Make sure your subtotal function includes order.addOns just like entrées, drinks, and desserts.

Example:

addItems(order.entrees, "entree");
addItems(order.drinks, "drink");
addItems(order.desserts, "dessert");
addItems(order.addOns, "add-on");
If your subtotal does not include add-ons, your receipt totals will not match the PHP result.

Step 6: Modify printReceipt()

In HW3, your function printed each line separately using console.log().

In this activity, you will change printReceipt() so it builds and returns one receipt string.

  • Create one string variable called receipt
  • Add lines to the string using +=
  • Use \n to move to a new line
  • Use \n\n if you want a blank line between sections
  • Use console.log(receipt) for debugging
  • Return the finished string using return receipt

Example:

let receipt = "";

receipt += "BYTE BISTRO RECEIPT - " + order.customer + "\n";
receipt += "Date: " + order.date + "\n\n";

// You may format the item lines however you prefer.
// The receipt must include items, quantities, subtotal, tax, tip,
// delivery fee if applicable, and total.

console.log(receipt);
return receipt;

Important: Your exact spacing and layout may look different from the example. That is okay. The important part is that your receipt includes the required information and the totals are correct.

Concept: Instead of printing step-by-step, you build the final result first and then display it.

Step 7: Add the Receipt Preview Section

You will display the receipt in a dedicated section in your sidebar. This section should be placed directly below your Live Order, so users can preview their receipt while reviewing their selections.

Example Structure

<section class="receipt">
  <h2>Receipt Preview</h2>

  <pre id="receiptOutput"></pre>

  <button type="button" id="previewBtn">Preview Receipt</button>
</section>

How it Works

  • The <pre> element displays the receipt text
  • The button will trigger your previewReceipt() function
  • The id="receiptOutput" must match your JavaScript exactly
  • You can add style to the receipt section using CSS

Why use <pre>?

The <pre> (preformatted text) element preserves spacing and line breaks.

This is important because your receipt is built as a multi-line string using \n:

receipt += "Item 1\n";
receipt += "Item 2\n";

Using a normal element like <div> would collapse the spacing, but <pre> keeps the formatting exactly as written.

Design Note:
The Preview Receipt button is not part of the form. It should be placed in this receipt section, not at the bottom of the form.
  • Select items → see them in the Live Order
  • Click Preview → immediately see the receipt below
This keeps the action and the result close together and avoids unnecessary scrolling.

Quick Check

  • Did you place the receipt section below your Live Order?
  • Did you include <pre id="receiptOutput"></pre>?
  • Is the id spelled exactly the same in your JavaScript?

Step 8: Write previewReceipt()

Now that your page has a receipt preview section, write the function that fills it.

The previewReceipt() function runs when the user clicks the Preview Receipt button.

function previewReceipt() {
  const order = buildOrderObject();

  const receipt = printReceipt(order, priceMap);

  document.getElementById("receiptOutput").textContent = receipt;
}
Big idea:
This function connects your data, calculations, and page display.

What each line does

const order = buildOrderObject();

Builds one complete order object from the current form values and selected menu items.

const receipt = printReceipt(order, priceMap);

Sends the order object to your receipt function. The receipt function calculates the totals and returns one formatted string.

document.getElementById("receiptOutput").textContent = receipt;

Displays the receipt string inside the <pre id="receiptOutput"> element you added in Step 7.

Data Flow

User selections + form inputs
        ↓
buildOrderObject()
        ↓
order object
        ↓
printReceipt(order, priceMap)
        ↓
receipt string
        ↓
receiptOutput
Important:
previewReceipt() does not submit the form. It only previews the receipt on the current page.

Step 9: Connect the Preview Button

Now connect your Preview Receipt button to the previewReceipt() function.

document.getElementById("previewBtn").addEventListener("click", previewReceipt);

This means:

  • When the user clicks the button
  • The previewReceipt() function runs
  • The receipt is displayed in the preview section
Important Make sure your button has id="previewBtn"and type="button".
